Version 1.0ß1        (30 April '98)

•        First version (unreleased).


Version 1.0        (08 May '98)

•        First 'official' version.


Version 1.5        (16 June '98)

+        Find option.

+        Print option.

+        Added speedbuttons for some functions.

+        Zoom option for current topic.

+        Copy current topic has been added to the 'File' menu.

+        Option for disabling the creation of indexfiles. (useful when viewing lists from a read only disk).

+        Commandline parameters: '/NOINDEX' to disable the creation of indexfiles and '/NOREGISTRY' to not modify the registry.

+        New option to enable/disable the automatic loading of the First topic when changing the list/category.

+        Added a statusbar to the configurationscreen which shows the pathnames of the comboboxes.

-        Bug: The current topic would be all in Bold font style when a new topic was selected and the cursor was placed on the first line of the previous topic. Fixed.

-        Bug: After the configuration form is closed with the 'Ok' button, the current loaded list was no longer marked. Fixed.

•        Better error handling when creating indexfile (When the indexfile cannot be created, the topics are still loaded).

•        When a list from the default location is loaded with 'Other', the actual list will be checked in the 'List' menu instead of 'Other'.

•        Default setting of Single Click Topic View is now 'On'.

•        Selected text of current topic remains selected when focus shifts to another component.

•        Several interface enhancements.


Version 1.5a        (22 June '98)

-        Bug: User couldn't select category '*' which was present in some lists (including INTERRUP.LST and PORTS.LST). Fixed.

-        Bug: When a list had unknown categories, the category title 'No categories available' wasn't displayed correctly. Fixed

-        Bug: When the user shifts focus away from Category ComboBox when the dropdown list is shown and the current category hasn't changed, the current category was reloaded. Fixed. 

•        Categories without descriptions are now mentioned as 'Unknown' instead of '?' in the Category combobox.

•        Improved detection of topics in a list when creating indexfile.


Version 1.5b        (08 July '98)

-        Bug: Categories with lowercase character wouldn't show up in Categories listbox (bug was only present in version 1.5a). Fixed.

-        Bug: Shortcut keys in Find dialog wouldn't work. Fixed.


Version 1.6        (23 July '98)

+        Added 'Match text case' option to Find dialog.

+        Added 'Whole words only' option to Find dialog.

+        Added option to load the last opened list on startup (in configuration dialog).

+        Added option to zoom topic with double click (and Enter-key) in config dialog.

+        Enhanced formatting of topics (things like 'Desc:' and 'SeeAlso:' are underlined, additionally, the formatting of topics can be disabled with a new option in the configuration dialog.

+        The plain text documentation of RBIL Viewer has been replaced by a help file (RBILVIEW.HLP) which can be accessed via 'Help' -> 'Help index…' (or F1). 

-        Bug: When the Find dialog is shown and closed with Cancel/Close button, text search starts from top when doing 'Search Next'. Fixed.

-        Bug: After Loading a list and changing the 'Location of list files' to a directory which contains no lists, 'Open...' will be checked in the 'List' menu. Fixed.

-        Bug: When a list was loaded for the second time with 'automatic creation of indexfiles' enabled AFTER it was loaded with 'automatic creation of indexfiles' disabled, the program would give an error message and corrupt the indexfile of the list. Fixed.

-        Bug: When option 'load first topic of list automatically' was ON, and a new list was loaded, the zoom option would be available when no topic was loaded which would result in an error while using zoom. When an other list was loaded, speedbuttons for 'Save', 'Copy', 'Print' and the popupmenu of the current topic would be enabled. All Fixed.

•        Loading a list (with indexfile) is up to 20% faster.

•        Changed Findtext input from 'Edit' to 'ComboBox' which has a list of the last used find strings.

•        Items of the topics list which are selected by a search operation now turn Black (instead of Teal), just as selected text of the current topic.

•        Topics with a length of 0 are ignored when creating an indexfile.

•        Redesigned Configuration dialog.

•        Redesigned Find dialog.

•        Esc-key is 'Unzoom' when viewing zoomed topic.

•        When loading a list with 'automatic creation of indexfiles' disabled, the progress dialog will now show an other message.

•        After loading a list (or creating a new indexfile), the current topic is automatically unzoomed (this also fixes some small bugs).


Version 1.6d        (22 September '98)

•        NOTE: This is only a maintenance update of v1.6, no major things were changed nor were any bugs fixed.

•        All colors within the program will change to match the current color scheme in Windows.

•        Open and Save dialog boxes can now be resized (in Win98).

•        Visual changes to Category Combobox.

•        The 'Browse' button for the 'location of lists' in the configurationscreen, will now display the standard Windows 95/98 directory selection dialog - instead of the custom one used in previous versions of the program.

•        [List Info] in Topics list is now in Bold text style.

•        Various small design changes.


Version 1.6e        (11 October '98)

-        Bug: Item in topics list, selected by a find operation would be all black. (bug was only present in v1.6d) Fixed.

•        List files under 'List' menu are only reloaded when the default directory of list files has actually been changed in the Configuration dialog.

•        When changing the Topic Text Formatting option in the Configuration dialog, the new setting immediately takes effect after closing the config dialog with the Ok button.


Version 1.61        (27 October '98)

+        New option for font size selection.

-        Bug: On some occasions the program would always create a new indexfile whenever a list was loaded, this bug was present since v1.0 and has now been fixed.  

•        Various small design changes.


Version 1.62         (04 December '98)

-        Fixed some minor incompatibility issues with release 59 of The Interrupt List.

-        Fixed some color settings for better support with different color schemes in Windows.

-        Unlike the 'Open-dialog', the 'Save-dialog' couldn't be resized in Win98, (although mentioned as a new feature in the help file of v1.6d). Fixed.

-        'Topic Title' column would remain black when selecting the 'Topic ID' column after a find. Fixed.

•        Changed the default setting for "Double click = Zoom topic" to Disabled.

•        Changed default column width of "Topic ID" for better visibility with a font size of 12.

•        Improved help file.


Version 1.62a         (24 January '99)

-        All prior 1.6xx versions wouldn't work if no printer was installed, this has been fixed in this version.

-        'Topic Title' column would remain black when selecting the 'Topic ID' column after a find. Mentioned as fixed in documentation of v1.62, but wasn't actually fixed in that version.

•        Should be a bit more stable on Windows 95 OSR1 (not tested though)

•        Zoom Topic is 'smoother'.


Version 2.0         (04 March '99)

+        Find function for finding a topic in any category regardless of the current one.

+        Topic cross referencing for topics in current list (with See Also menu).

+        Print Selection option.

+        Tools menu. 

+        Category character is displayed when viewing 'All categories'.

+        User-defined category descriptions now possible (with CATDESC.INI).

+        Changeable Font styles and Sizes.

•        Changed menu layout.

•        Internet Explorer-like toolbars.

•        'Find' shows loads the found topic directly in zoomed mode, even when 'Load selected topic' is OFF.

•        Many more configurable options in configuration screen.

•        Loading topic titles (opening list) up to 50% faster.

•        Loading (showing) topics 50% to 95% faster.

•        Changing categories is a lot faster.

•        Finding text is a bit faster.

•        Better detection of 'real' topic titles.

•        Topics which contain no data are no longer shown.

•        OPCODES.LST is parsed much better.

•        Improved support for legacy lists and plain text files.

•        The program is overall more stable after an error occurs.

•        All colors within the program use the current Windows color scheme.

•        Screen layout also looks good with Large Fonts setting in windows.

•        Better Help file.


Version 2.0 r2        (07 March '99)

-        Fixed memory leak bug in indexfile creation routine.


Version 2.1        (05 April '99)

+        Hot Links for topic cross referencing.

-        Current list would no longer be checked after a 'Update Topic Titles' command. Fixed.

-        Cross referencing failed for some quite some topics in I2C.LST. Fixed.

-        Clicking on the first column of the Topic Titles list after a list was closed would display 'Sel: 1' on the statusbar. Fixed.

-        Things like '!!!ftp://' on the start of a line was underlined when topic formatting is on. Fixed. 

-        'Search again' in topic titles wouldn't start from top when a new list was loaded. Fixed.

-        'Search again' in topic text didn't start from top when a new topic was shown. Fixed.

-        Main Toolbar would always be aligned to the right side of the screen when using the program on a system with Internet Explorer 5 installed. Fixed.

•        A lot of internal optimizations

•        Loading indexfiles is a lot faster, progressbar only shows up on lists which have more than 1000 topics. Optimized indexfile creation routine.

•        Improved cross-referencing. Cross-referencing now works in about 95% of all cases.

•        The program now uses the Richedit 2.0 component to display the current topic (is also still compatible with Richedit 1.0).

•        Category Combobox is now resizable so longer category descriptions will also fit.
